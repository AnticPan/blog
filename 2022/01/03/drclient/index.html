Linux下配置DrClient并自动化连接

2022-01-03
  • Linux
  • Network

# 背景

上网需要使用DrClient客户端验证，原本使用的Dr版本为1.0，在Linux可以正常使用，但在更新后要使用1.1.5版本的Dr，并且存在依赖缺失问题，无法直接使用。在此记录依赖修复过程和自动化联网方法。

# 目标

  1. 修复执行DrClientLinux报错问题，报错如下：
./DrClientLinux: error while loading shared libraries: libssl.so.10: cannot open shared object file: No such file or directory
1
  1. 实现Dr断网自动重连

# 测试环境

  1. Manjaro 21.2rc Qonos
  2. Ubuntu 16.04

下面仅以Manjaro为例，Ubuntu仅需将/usr/lib/路径改为/usr/lib/x86_64-linux-gnu/。

# 修复过程

  1. 使用ldd命令查看新版DrClientLinux的依赖, 发现缺少libssl.so.10和libcrypto.so.10两个依赖
......
libX11.so.6 => /usr/lib/libX11.so.6 (0x00007fabefb3f000)
libssl.so.10 => not found
libcrypto.so.10 => not found
libz.so.1 => /usr/lib/libz.so.1 (0x00007fabefb25000)
......

  1. 经过搜索，缺失的两个依赖是Centos下拥有的。经过尝试，进一步定位到openssl-libs-1.0.2k这个包
  2. 下载 (opens new window)并解压这个包，在openssl-libs-1.0.2k-21.el7_9.x86_64/usr/lib64/目录下找到两个缺失依赖
  3. 将两个原始文件拷贝到/usr/lib目录下（.so.10文件只是链接），并重新加载系统依赖库
sudo cp libcrypto.so.1.0.2k /usr/lib
sudo cp libssl.so.1.0.2k /usr/lib
sudo ldconfig

# 自动化联网

  1. 确保可以运行DrClientLinux并勾选保存密码和自动登录
  2. 安装xvfb-run，用于虚拟X环境来在后台运行DrClientLinux
  3. 创建脚本AutoReconnect.sh，将下方[DrClient Dir]改为Dr文件夹的绝对路径
#!/bin/bash

ping_state=`ping 114.114.114.114 -c 5 | grep "[1-5] received"`
echo `date` $ping_state
if test -z "$ping_state"
then
    echo "start kill Dr"
    ps -ef | grep 'xvfb' | grep -v grep | awk '{print $2}' | xargs kill -9
    ps -ef | grep 'DrClient' | grep -v grep | awk '{print $2}' | xargs kill -9
    cd [DrClient Dir]
    xvfb-run ./DrClientLinux &
    echo "finish restart Dr"
fi
